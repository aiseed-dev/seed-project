# デザイン原則(seed-project)

視覚言語の正は `docs/reps/seed-v1/docs/10_design.md`(種苗カタログ様式)。
このスキルはそれを実装規則に落としたもの。矛盾したら docs/10 が正。

## 0. 基本(docs/10 の要点)

- 配色は core の `SeedColors` のみ(Widget内での直接色指定は禁止)。
  背景 #F5F1E6 / カード #FFFFFF / 文字 #33302A / 緑 #5A7A2E /
  橙 #C0761A / 青 #3A6B8A / 無効 #A9A297
- 主ボタンは橙のみ、**1画面に1種類の強調**
- 出品カードは core の `ListingCard`(解剖は docs/10)を全アプリで共通使用
- 文言は動詞から・感嘆符/絵文字を使わない・状態表示は丁寧語

## 1. レスポンシブデザインの原則

「画面を2重に作らない」が大前提。**同じ画面Widgetの中で**レイアウトだけを
出し分ける。

### 1-1. ブレークポイント(全アプリ共通)

| 名前 | 幅 | ナビ | 一覧 | 事務機能(CSV・エクスポート) |
|---|---|---|---|---|
| モバイル | < 600px | 下部ナビ | カード2列 | 非表示(メニューに「Web版で利用できます」) |
| タブレット | 600–899px | 下部ナビ | カード3列 | 非表示 |
| ワイド | ≥ 900px | 左ナビ(NavigationRail) | カード4列 / DataTable | 表示 |

- 境界値は `900` を唯一の「業務用ワイド」境界とする(apps/shop の
  `wideBreakpoint`)。600 はグリッド列数の調整のみに使う
- 数値をリテラルで散らばらせない。アプリの `config.dart` の定数を参照する

### 1-2. 実装ルール(Flutter)

1. **LayoutBuilder で出し分ける。** 画面全体の判断は `MediaQuery.sizeOf`
   でも良いが、部品単位の判断は必ず `LayoutBuilder`(親の制約基準)。
   埋め込まれた場所によって正しく縮退するため
2. **グリッドは列数ではなくカード幅で考える。** 実質カード幅が
   160–240px に収まるよう列数を決める。HTML では
   `grid-template-columns: repeat(auto-fill, minmax(220px, 1fr))`
3. **フォームは幅制約。** 入力フォームは `ConstrainedBox(maxWidth: 420)`
   程度に制約し、ワイド画面で入力欄が間延びしないようにする
4. **表は横スクロール。** DataTable がはみ出す場合は画面全体ではなく
   その表だけを `SingleChildScrollView(scrollDirection: horizontal)` で
   スクロールさせる
5. **機能を隠すときは案内を残す。** モバイルで CSV 取込等を消すときは、
   メニューに「Web版(広い画面)で利用できます」の無効項目を残す
   (機能の存在を知らせる)
6. **タップターゲットは 44×44px 以上。** アイコンボタンの密集を避ける
7. **はみ出しを構造で防ぐ。** 固定高さカード内の可変テキストは
   `Expanded` + 非スクロール `SingleChildScrollView` 等で「静かに切り詰める」
   (RenderFlex overflow を出さない)。ListingCard の実装が手本

### 1-3. kit(カタログHTML)

- ページ全体の横スクロールは禁止。幅のある要素(表)は要素内スクロール
- 画像・カードは `max-width: 100%`。グリッドは auto-fill + minmax

## 2. 文字サイズとユーザー設定(スケーリング規則)

文字は2種類に分けて扱う。**混ぜないこと。**

### 2-1. 「読む文字」= ユーザー設定に従う

対象: 説明文・辞典記事・メッセージ本文・規約などの**文章**。

- OS の文字サイズ設定(`textScaler`)に**必ず追従させる**
  (アクセシビリティ。高齢の利用者が多い前提)
- 行の高さ・余白は拡大されても破綻しないよう、固定高さの器に入れない

### 2-2. 「デザインの文字」= 設定の影響を受けない

対象: **デザイン上、意図してサイズを決めている文字**。具体的には

- 価格(カード・詳細の橙の金額)
- 種別バッジ(販売/交換/譲渡)・チップ(固定種/在来種/自家採種)
- キャッチコピー(緑・小)・カードのスペック行(ラベル: 値)
- 統計数値(ダッシュボードの件数)・ナビのラベル

これらは**サイズオプション(textScaler)にもスタイルオプション
(太字設定 boldText、テーマの DefaultTextStyle)にも影響されない**。
理由: カタログ様式は「同じ位置・同じ様式での対照」(業者品の発芽率と
個人品の無保証注記など)が情報設計の核であり、文字が拡大されて折り返す
とカードの対照が壊れる。読む情報は 2-1 の文章側と詳細画面で保証する。

実装(Flutter):

```dart
// core の DesignText を使う(textScaler固定+inherit:false)
DesignText('360円', size: 15, color: SeedColors.orange, bold: true)

// 生の Text で書く場合は必ず両方を指定する
Text('販売',
  textScaler: TextScaler.noScaling,             // サイズ設定を無効化
  style: TextStyle(inherit: false, ...))         // 周囲のスタイルを継承しない
```

- core の共通Widget(TypeBadge / SeedChip / ListingCard の価格・スペック行)
  は DesignText 適用済み。**新しいバッジ・価格表示を作るときも DesignText**
- カード全体を `MediaQuery.withNoTextScaling` で包む方法は、本文
  (タイトルや説明)まで固定してしまうため**使わない**。固定するのは
  デザイン文字だけ

実装(HTML / kit カタログ):

- デザイン文字(価格・チップ・バッジ)は `px` 指定
  (ブラウザの文字サイズ設定に追従させない)。本文は `rem`
- **ブラウザのページズームは常に許容**(全体が等倍で拡大されるのは壊れない)

### 2-3. 検証

- ウィジェットテストで `MediaQuery(textScaler: TextScaler.linear(2.0))`
  を掛けてカード一覧を描画し、**RenderFlex overflow が出ないこと**と
  **価格・バッジのサイズが変わらないこと**を確認する
- 逆に、説明文・記事本文が 2.0 倍で拡大されることも確認する(追従の保証)

## 3. 変更時のチェックリスト

- [ ] 色は SeedColors 経由か(直接色指定なし)
- [ ] 橙の主ボタンは画面に1種類か
- [ ] < 600 / ≥ 900 の両方でレイアウトを確認したか(画面の2重実装なし)
- [ ] モバイルで隠した機能に案内を残したか
- [ ] 価格・バッジ・スペック行は DesignText(textScaler固定)か
- [ ] 本文・説明は textScaler に追従するか
- [ ] textScaler 2.0 でオーバーフローしないか
