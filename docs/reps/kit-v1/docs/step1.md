# 12 ステップ1: 軽量キット(カタログHTML+注文の神Excel+請求書)

本体アプリ(Phase 1〜)に**先行して**作る最初の一歩。DB・サーバー・
アプリなしで、店(たねの森を想定)の今の運用を一切変えずに機械可読に
する。担当者(または当方)のPCだけで完結する。すべて Python。

## 3点セット

**(1) パンフレット(カタログ)の HTML 化**
- 店が維持する**商品台帳 xlsx**(神Excel様式: 品番・品種名・種別
  〔固定種/在来種/F1 のドロップダウン〕・価格・説明・生産地・発芽率・
  在庫状態・カテゴリ)を openpyxl で読み、カタログ HTML を生成
  (カテゴリ別一覧+品目ページ。カードの様式は docs/10 を簡略適用)
- Cloudflare Pages または店の既存サイトに置く。手更新の在庫表・
  PDF カタログを置き換える

**(2) 注文用の神Excel**
- 商品台帳から**注文様式 xlsx を自動生成**(品番は入力規則で選択、
  数量、氏名・住所・電話・メール欄、名前付きセルで読み取り位置固定、
  記入例シート付き。**印刷すればそのまま FAX 注文書**)
- カタログからダウンロード+メール添付で配布。顧客は記入して
  メール添付で返送(または FAX・郵送)

**(3) 請求書作成**
- 受信した注文 xlsx を所定フォルダに入れて実行すると:
  読み取り(名前付きセル)→ **注文台帳 xlsx** に記帳(注文番号=
  年+連番を採番)→ **請求書 xlsx** を生成(品目・数量・単価・合計・
  振込先。税表示は店の課税状況に合わせて設定)
- 後払い(郵便振替等)という現行の商流のまま。決済は作らない
- 読み取れないファイルは「未処理」フォルダへ移して人が見る
  (黙って捨てない)

## 原則

- **商品台帳 xlsx(店の手元)が正。** カタログ HTML・注文様式・
  (将来の)店側在庫 API はすべてここから派生する。
  「aiseed は在庫を持たない。在庫の正は店側」(docs/01 S-08)と整合
- 様式の設計方針(名前付きセル・入力規則・保護・記入例・FAX 兼用)は
  本体の様式取込と共通にする
- コードは kit/ ディレクトリに置き、Forgejo で管理。この規模の
  スクリプトはローカルのコーディングモデル(North Mini Code 級)でも
  生成・改修できる

## 位置づけ

- **たねの森への最初のデモ・導入物。** 提案書ではなく動くものを見せる。
  数日で作れて、店の手作業(在庫表の手更新・注文の転記・請求書の手書き)
  が目の前で消える
- ステップ1で信頼ができてから、本体(交換・コミュニティ・辞典=
  Phase 1〜)へ進む。ステップ1の様式・パーサ・請求書生成は、
  本体の shop 経路(様式メール取込)と店側在庫 API にそのまま昇格できる
