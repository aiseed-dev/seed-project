# 05 販売用アプリ(Flutter)画面仕様

対象: iOS / Android / Web(+ Windows / macOS)。店舗スタッフが使う。
**AGPL-3.0・無料**(LICENSING.md 参照)。
単体でも使えるが、種の森(統合アプリ)への出店プラン(定額)が主導線。

**実装場所**: 画面はこのアプリ内(`apps/shop`)に自己完結で実装する
(アプリ独立の原則)。共有は core(MIT)の配管のみ。
実装パターン(自己完結型Widget)は交換用アプリと共通。

## レスポンシブ方針(スマホ対応の要)

画面幅 900px を境に2レイアウト。**同じ画面Widgetの中で LayoutBuilder により
出し分ける**(画面を2重に作らない):

| 機能 | モバイル(<900px) | ワイド(≥900px: Web/デスクトップ) |
|---|---|---|
| ナビ | 下部ナビ(ホーム/出品/申込み/設定) | 左ナビ |
| 出品管理 | カードリスト+長押しで複数選択 | DataTable+チェックボックス |
| 一括操作 | 選択時に下部アクションバー | 選択時に上部アクションバー |
| CSV取込 | **非表示**(メニューに「Web版で利用できます」) | 表示 |
| エクスポート | **非表示**(同上) | 表示 |
| 申込み対応 | フル機能(スマホの主用途) | フル機能 |

スマホ版の主目的は**外出先での申込み対応と在庫確認**。
事務作業(CSV・エクスポート)は画面の広い環境に限定する。

## ログインと店舗コンテキスト

- PocketBase ログイン(利用者と同一アカウント基盤)。認証は個人別なので、
  同一店舗に複数スタッフが各自のアカウントで所属できる
- GET /me → shop_members 所属を確認。所属なしなら「店舗アカウントが
  必要です」画面(運営への連絡先)。1ユーザー1店舗
- 初回ログイン後、自分の担当部門/担当者名(contact_label)を登録する
  (例「種苗部 田中」)。後から店舗設定でも変更可
- 出店プラン(定額利用料)の有効性は shops.is_active で判定
  (契約・請求はアプリ外。決済コードは書かない)

## 画面

### ダッシュボード(ホーム)
- 公開中出品数 / 今週の申込み数 / 未対応(requested)件数(赤バッジ)
- 直近の申込み5件(タップで詳細へ)

### 出品管理
- フィルタ: 状態(公開中・停止・終了)・分類
- 項目: サムネ / 品種名 / 種別 / 価格 / 在庫(店側在庫APIの値) / 状態 /
  閲覧数 / 申込み数 / 更新日
- 在庫は aiseed では持たない。店側在庫APIの値を表示するのみ
  (在庫の管理・更新は店側システムの責任)
- 一括操作: 公開 / 停止 / 価格変更(ダイアログ)
- 新規出品・編集フォームは docs/04 の出品フォームと同じ構成を実装
  (生産物・受け渡し・支払い既定・食品表示・特商法表示を含む)
  (既定値のみ「販売」に変更。確認チェックも同様に必須)

### CSV取込(ワイドのみ)
- テンプレートDL(ヘッダ: 品種名,分類slug,種苗区分,種別,価格,数量表記,
  採種年,生産地,発芽率,種子消毒,説明,栽培メモ)。
  店舗出品は requires_seed_label=true 固定。表示者氏名住所は店舗
  プロフィールから自動補完(CSVに列は不要)
- POST /shop/listings/import → 結果テーブル:
  緑=作成 / 黄=作成+品種マスタ提案を生成 / 赤=エラー(理由列)
- 「黄は運営の承認後に辞典と紐づきます」注記
- 指定種苗の表示事項(生産地・種子は発芽率)が欠けた行は赤エラー
- 取込後「QRラベル一括PDF」ボタン: 各出品のQR(品種名つき)を面付けした
  PDFを生成(API側)。種袋・店頭棚に貼る

### 申込み対応
- 店舗宛の全申込み(requested を上に)。スタッフ全員で共有、開いたら既読
- 各申込みに**申込番号**(DB全体の通し番号、例 2026-00042)を表示。
  経理・外部販売システムとの照合に使う
- 承諾すると承諾担当者(自分の contact_label)が記録され、
  スタッフ間で「誰が対応したか」が分かる
- 申込み詳細: 申込番号・品目リスト・数量・小計 → 承諾/辞退 → メッセージで
  支払・発送のやり取り → 完了(成約日が記録され経理エクスポートに載る)
- 在庫の増減は店側システムの責任(aiseed は申込みを店側へ通知するのみ)

### エクスポート(ワイドのみ)
- 種類(出品一覧 / 成約台帳)×形式(xlsx/CSV)→ DL
- 成約台帳: 申込番号・店舗コード・受付日・承諾日・成約日・相手・品目・
  数量・金額。番号は DB全体の通し番号。店の販売・会計システムの伝票番号と
  対応づけられる(連携方式は店ごとに決定)
- 「xlsx は OnlyOffice・LibreOffice でそのまま開けます」注記

### 店舗設定
- 自分の担当部門/担当者名(contact_label)の編集(全スタッフ)
- 以下は owner のみ: 店名・説明・ロゴ・サイトURL・地域・
  特商法表示用の連絡先・返品方針・引き渡し時期
- 店舗コード(経理エクスポートの識別列)は運営が設定(表示のみ)
- スタッフ一覧(担当名つき。追加は運営に依頼する旨。当面 admin 経由)

## 実装メモ

- モバイルの file picker: image_picker、デスクトップ/Web の CSV 選択:
  file_selector(許可パッケージ)
- テーブル/リスト系も自己完結型Widgetパターン(fetch・ページング・
  選択状態を自Widgetで保持)
