# 種の交換アプリ プロジェクト一式

固定種・在来種の種苗を個人間・店舗で交換・販売できるコミュニティアプリ群。
伝統野菜辞典アプリと品種マスタを共有し、出品行為が辞典を充実させる構造を持つ。

統合利用者アプリは採用が決まったら名称を「種の森」に変更予定。
セールス先: たねの森 (tanenomori.org) / 自然のタネ (natural-harvest.ocnk.net) /
野口のタネ(出店プラン提案の第一候補)。

## 3つの製品アプリ+内部ツール

| アプリ | 対象 | ライセンス | 配布 |
|---|---|---|---|
| 交換用アプリ | 個人(交換・販売・譲渡) | AGPL-3.0+ストア配布許可(無料) | iOS・Android・Web |
| 販売用アプリ | 店舗(出品管理・申込み対応) | AGPL-3.0+ストア配布許可(無料) | iOS・Android・Web(+デスクトップ) |
| 統合利用者アプリ | 自社運営「種の森」(**品目軸**で交換・辞典・料理〔将来〕を統合) | 非公開 | iOS・Android・Web |
| 管理アプリ | 運営ツール(Flet/Python・core非依存) | MIT(公開) | デスクトップ/ローカルWeb |

**3つの Flutter アプリは別のソフトとして独立実装**し、共有は core
(MIT: APIクライアント・モデル・共通Widget・配色)の配管のみ。
この分離により、AGPL 側は PR を自由に受けつつ統合アプリは非公開を
維持できる(CLA 不要)。詳細は `LICENSING.md` と `CLAUDE.md`。

## 技術スタック(確定済み)

- バックエンド: Python / FastAPI / SQLAlchemy 2.0 / PostgreSQL
- 認証: PocketBase(身元のみ。業務データは持たせない)
- メール: Stalwart(自宅サーバー・固定IP・直接送信)
- フロント: Flutter(Riverpod/provider 不使用。小さな自己完結型Widget)
- 公開: Cloudflare プロキシ(Full Strict)+ Caddy。Tunnel は使わない
- 画像: ローカルディスク保存 + Cloudflare エッジキャッシュ + rclone夜間バックアップ

## 設計上の要点

- **種苗法対応が最大のセールスポイント**: (A)登録品種の出品をDB制約とUI
  両方でブロック(固定種・在来種のみ)。(B)指定種苗の表示義務(22条)に
  対応し、種苗業者(店舗等)の出品に氏名住所・生産地・発芽率等の表示欄を
  必須化。店舗は本アプリを使うだけで指定種苗表示に対応できる。
  (C)正規業者品(発芽率あり)と個人の家庭採種品(無保証)を同じ様式で
  対照表示し、買う人が一目で区別できる。
- **出品が辞典を育てる**: マスタ未登録の品種名で出品すると品種マスタへの
  提案が自動生成され、承認されると辞典記事枠ができる。
- **取扱いと支払い**: 種・苗・生産物(生鮮野菜)を扱う(加工品・許可要食品は
  対象外)。受け渡しは直接/郵送。支払いは後払いを原則とし前払い等も当事者
  協議で可(決済は持たない)。郵送の生産物は食品表示、郵送の事業者は特商法表示。
- **在庫・申込番号・経理**: aiseed は在庫を持たない(在庫の正は店側。
  店側在庫APIから取得して表示のみ)。全ての申込みに申込番号を
  採番(DB全体の通し番号、年+連番)し、成約を経理台帳としてエクスポート
  (店の販売システムと照合可能)。DBを分離すればその中で一連になる。
- **カート+申込み方式・決済なし**: カートは提供者ごとにグループ化し、
  申込み単位で取引(承諾→メッセージで支払・受渡し合意→完了→相互評価)。
  金銭処理はアプリ外。**送料は交換以外(販売・譲渡)は別**(実費を
  メッセージで合意)。交換は各自が自分の発送分を負担。
  店舗の出店プラン(定額)・保守契約もアプリ外(請求書・銀行振込)。
- **事後審査**: 出品は即公開、通報ベースで対応。辞典編集のみ事前承認制。
