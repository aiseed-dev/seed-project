# 01 機能要件

## 対象と目的

固定種・在来種(種苗法上の「一般品種」)の種・苗を、個人および販売店が
交換・販売・譲渡できる場を提供する。登録品種は扱わない。
伝統野菜辞典と品種マスタを共有し、コミュニティの活動が辞典を充実させる。

## ユーザー種別

| 種別 | 説明 | 使うアプリ |
|---|---|---|
| 一般ユーザー | 個人。出品・交換・辞典編集提案 | 交換用 or 統合利用者アプリ(種の森=品目軸) |
| 店舗スタッフ | shops に所属。店舗名義で出品 | 販売用アプリ or 統合アプリの店舗モード |
| editor | 辞典リビジョンの承認(**当初から必要**。外部協力者可) | Flutterアプリ内の承認画面(editor API 経由) |
| 運営者(moderator/admin) | 通報対応・品種承認・店舗/ユーザー管理 | 管理アプリ(Flet・DB直結・サーバー上で実行) |

editor に DB 認証情報は渡さない。承認画面は role=editor 以上にのみ表示される。

機能IDの読み替え: F-xx(交換系)・S-xx(店舗系)は各アプリ内に
自己完結で実装する(共有は core の配管のみ)。

## 機能一覧

### 交換機能(seed アプリ・aiseed アプリ)

- F-01 会員登録・ログイン(PocketBase。メール確認必須)
- F-02 ホーム: 分類グリッド(8分類)+ 新着出品の横スクロール
- F-03 出品一覧: 分類別・カード型(写真/品種名/交換・販売・譲渡バッジ/地域/店舗バッジ)
- F-04 検索: 品種名(かな・別名対応)、分類・種別(交換/販売/譲渡)・地域で絞込
- F-05 出品詳細: 写真スワイプ、品種情報、採種年、自家採種フラグ、
      栽培メモ、出品者プロフィール、辞典記事へのリンク
- F-06 出品: 写真(最大4枚)→分類→品種名(マスタからインクリメンタルサジェスト、
      無ければ自由入力=マスタ提案を自動生成)→種/苗→交換・販売・譲渡→
      条件(希望品 or 価格)→採種年・地域→**登録品種でないことの確認チェック(必須)**
- F-07 カート: 全種別共通「カートに入れる」。カート内は提供者
      (店舗/個人)ごとにグループ表示。グループ単位で申込みを送る
- F-08 申込みと取引: 申込み(品目リスト+一言)→承諾/辞退→
      メッセージ(住所交換・支払合意はここ)→双方完了→相互評価。
      新着はメール通知(未読が続く場合のみ、連投抑制)
- F-08a 送料: **販売・譲渡は送料別**(実費。金額と支払方法は申込み後の
      メッセージで合意)。**交換は各自が自分の発送分を負担**
      (送料の授受なし)。アプリは送料計算機能を持たず、
      表示上の注記(「送料別」)と合意の場のみ提供する
- F-09 マイページ: 出品管理(編集・取引中・終了)、評価一覧、辞典への貢献履歴
- F-10 通報(出品・メッセージ・ユーザー)
- F-11 辞典連携: 出品詳細から品種記事へ。記事の「編集を提案」で
      セクション単位の編集→承認待ちに入る

### 店舗機能(shop アプリ・aiseed アプリ。詳細は docs/05)

- S-01 店舗プロフィール管理(店名・説明・ロゴ・サイトURL)
- S-01a 担当者情報: 各スタッフが自分の担当部門/担当者名(contact_label)を
      登録・編集(自由入力)。認証は個人別で、同一店舗に複数の担当者が
      所属できる。申込み対応の表示・記録に使う
- S-02 店舗名義の出品(既定は「販売」。個人と同じ法確認チェック)
- S-03 CSV 一括出品(テンプレート提供。品種名はマスタ照合、
      未登録分はまとめて提案生成)
- S-04 出品の一括操作(公開/停止/価格変更)
- S-05 申込み受信箱: 店舗宛の申込みをスタッフ全員で共有・対応(承諾/辞退/メッセージ)
- S-06 エクスポート: 出品一覧・取引履歴を xlsx/CSV で出力(OnlyOffice で開く前提)
- S-07 簡易統計: 出品ごとの閲覧数・申込み数
- S-08 在庫: **aiseed は在庫を持たない。在庫の正は店側。**
      店側に在庫API(標準HTTP+JSON。品目ごとの在庫を返す最小API)を実装し、
      aiseed はそれを取得して表示する。テスト段階は正確な数を表示、
      表示粒度(内数: あり/わずか/なし等)は後日決定。ずれは申込み時に
      当事者間で吸収(後払い原則のため許容できる)。個人出品は在庫概念なし
- S-09 申込番号: 全ての申込みに DB全体の通し番号(年+連番、例 2026-00042)を
      採番。経理・外部販売システムとの照合キー。DBを分離すればその中で
      改めて一連になる。成約台帳にエクスポート(店舗コード列で店を識別)
- S-10 経理エクスポート: 成約(completed)を申込番号・受付/承諾/成約日・
      相手・品目・数量・金額で xlsx/CSV 出力(自店のみ)

### 管理アプリ

- A-01 通報キュー(open のみ、古い順)。対応: 出品停止・ユーザー凍結・却下
- A-02 品種マスタ承認キュー(pending)。承認で辞典記事枠を自動作成。
      登録品種の疑いがあれば農水省品種登録DBで照合し registration_note に記録
- A-03 辞典リビジョン承認キュー(差分表示付き)
- A-04 店舗管理: 店舗の作成・認証バッジ付与・スタッフ紐付け
- A-05 ユーザー管理: 検索・凍結・役割変更
- A-06 ダッシュボード: 出品数・成約数・辞典記事数の推移

## 非機能要件

- 可用性: 短時間のダウンは許容。計画停止は事前告知のみ
- バックアップ: DB・画像を毎晩オフサイトへ(RPO 24時間)
- 決済: 実装しない。金銭授受はユーザー間の責任(利用規約に明記)
- 個人情報: 住所はDBに保存しない(メッセージ本文中でのみ交換される)。
  メッセージは当事者と moderator 以上のみ閲覧可
- メール: 登録確認・メッセージ着信・承認結果の通知。Stalwart から送信

## 種苗法対応(必須要件)

### (A) 育成者権 — 登録品種の出品禁止
1. 出品フォームに「本品は品種登録されていない一般品種(固定種・在来種)です」
   のチェック必須(DB制約でも担保済み)
2. 品種マスタに is_registered_variety=true が付いた品種は出品不可
   (API でバリデーション、既存出品は moderation='flagged' に自動変更)
3. 利用規約に登録品種出品禁止・違反時の出品削除を明記
4. 通報理由に「登録品種の疑い」を用意し、管理アプリで優先表示

### (B) 指定種苗の表示義務(種苗法22条)
指定種苗(穀類・豆類・いも類・野菜等の食用作物、花き、果樹、芝草の
ほぼ全て)を「業として」販売する者=種苗業者(店舗、および反復継続的に
販売する個人)は、包装表示または証票添付が義務。オンライン広告にも
表示が必要。**個人の家庭的な交換・少量の譲渡は対象外。**

5. 出品が表示義務の対象か(requires_seed_label)を判定する:
   - 店舗出品(shop_id あり)は既定 true
   - 個人出品は既定 false。ただし出品フォームで「反復継続して販売している
     (種苗業者に該当)」を選ぶと true
6. requires_seed_label=true の出品は次を必須入力とする(API 422で検証):
   表示者氏名/名称・住所、生産地(都道府県)、種子は発芽率も。
   苗は発芽率不要。種子消毒等の処理は任意(あれば表示)
7. 表示欄は出品詳細・カードに「指定種苗表示」として明示する
   (店舗の氏名住所はプロフィールから既定値を補完してよい)
8. 免責: 自分が種苗業者に当たるかの最終判断は出品者の責任。
   利用規約に明記し、アプリは表示欄の提供と促しを行う

### (C) 業者品と個人品の区別を明示する
正規の種苗業者(店舗等)と個人の家庭採種品が同じ場に並ぶため、
両者の性質を一目で区別できるようにする(誠実さとトラブル予防)。
9. 個人出品(requires_seed_label=false)は no_warranty を既定 true とし、
   「家庭採種・発芽保証なし」をカード/詳細に明示する
10. 見分け線は指定種苗表示の有無=発芽率の有無。業者品は発芽率あり、
    個人品は発芽率なし+無保証注記、という対照がスペック行の同じ位置で
    見えるようにする(docs/10)
11. no_warranty は表示義務の回避手段ではなく個人出品の性質表示である旨を
    利用規約に明記する

※(B)は登録品種か一般品種かと無関係に、指定種苗を業として売るなら生じる
義務。本アプリは(A)で登録品種を除外済みだが、一般品種でも(B)は必要。

## 取扱品目と受け渡し・支払い

- 品目種別(item_kind): 種 / 苗 / 生産物(生鮮野菜)。
  **加工品・許可要食品は対象外**(将来の別ドメイン。設計思想は docs/11)
- 受け渡し(delivery_method): 直接(直売・手渡し・取り置き)/ 郵送
- 支払い(payment_default): **後払いを原則**(既定)。前払い・着払いも
  当事者協議で可。方法・金額・タイミングはアプリ外で当事者が決める。
  アプリは既定表示と規約での注意喚起のみ(決済機能は持たない)

## 生産物・郵送時の表示義務

- (D) 食品表示: 生産物(生鮮野菜)を郵送するとき、名称・原産地・生産者・
  収穫日・保存方法を表示。直接受け渡しは対面のため軽い(口頭で足りる)
- (E) 特定商取引法の販売者表示: 郵送で業として売る事業者(店舗、事業者的
  個人)は、氏名/名称・住所・連絡先・販売価格・送料・支払方法と時期・
  引き渡し時期・返品条件を表示(requires_tokushoho)。店舗は
  プロフィール〔住所・連絡先・返品方針・引き渡し時期〕から補完。
  個人の家庭的な直接受け渡し・少量譲渡は事業者に当たらず対象外
