# LICENSING — ライセンスと収益の方針(最終版)

**アプリ独立の原則により、CLA・リンク例外は不要になった。**
公開コードへのプルリクエストは自由に受けられ(inbound=outbound)、
統合アプリは非公開のまま維持できる。

## 一覧

| 構成物 | ライセンス | 公開 | 費用 |
|---|---|---|---|
| backend/(FastAPI)、db/schema.sql | AGPL-3.0-only | 公開 | 無料 |
| packages/core(配管: APIクライアント・モデル・共通Widget・配色) | **MIT** | 公開 | 無料 |
| apps/seed(交換用アプリ) | AGPL-3.0-only **+ストア配布の追加許可** | 公開 | 無料 |
| apps/shop(販売用アプリ) | AGPL-3.0-only **+ストア配布の追加許可** | 公開 | 無料 |
| admin/(管理アプリ・Flet/Python) | MIT | 公開 | 無料 |
| aiseed(統合アプリ「種の森」) | プロプライエタリ | **非公開** | 個人無料 / **店舗は定額利用料** |
| 辞典コンテンツ(ユーザー投稿記事・写真) | CC BY-SA 4.0 | 公開 | — |

## この構成が成立する理屈

1. **統合アプリは独立実装**であり、公開アプリのコードを含まない。
   依存する公開コードは core(MIT)のみ。MIT のコードは外部貢献が
   混ざっても非公開アプリで自由に利用できる
2. よって **AGPL 側(backend / 交換用 / 販売用)は CLA なしで PR を
   受け入れてよい**(貢献は同じライセンスで受ける)。貢献が増えても
   統合アプリに影響しない
3. **コピー禁止則**(CLAUDE.md 参照): 外部貢献の受け入れ開始後は、
   公開アプリのコードを aiseed へコピーしない。
   これが唯一守るべき規律である
4. **AGPL の防壁はサーバー側で機能する**: backend を流用してサービスを
   立てる者は改変の公開義務を負う。クローン SaaS 防止の目的は
   backend の AGPL だけで達成される

## ストア配布の追加許可(AGPLv3 第7条)

交換用・販売用アプリの LICENSE に以下の趣旨の追加許可を付す
(App Store 等の配布規約と AGPL の衝突を避けるため。文面は
専門家レビュー前提のたたき台を用意する):

> 追加許可として、本プログラムの改変の有無にかかわらず、
> アプリケーションストアの配布条件に従うことを目的とする限りにおいて、
> AGPL-3.0 第6条の要求と矛盾するストア側の条件があっても、
> 対応するソースコードを本リポジトリで公開し続けることを条件に
> オブジェクトコード形式で配布することができる。

この許可は AGPL-3.0 §7 に基づき削除可能(受領者は純粋な AGPL として
扱ってもよい)。

## 収益モデル(ソフトは売らない。サービスを売る)

料金は誰に対しても同一(定価制)。**取引手数料は取らない**
(店舗は定額のみ。売上に連動しない。金額把握=決済に近づくため)。

- **個人: 無料**(年会費なし。数と活気を集める)
- **店舗のアプリ利用料: 月1万円** — 種の森(aiseed)への出店。
  shop から aiseed へは「出店」という形で参加する
- **運用支援: 月1万円(任意)** — 必要な店だけが契約。システムの正常稼働
  維持とバックアップの支援まで。契約しない自力運用も自由(AGPL)
- **コミュニティ支援: 時給5千円(一律)** — 導入時のデータ投入、
  使い方の案内、出品・辞典まわりの軽微な調整。**初心者水準で担える
  作業に限る**(運営者の専門時間を前提にしない。将来は講習を受けた
  地域の協力者や店舗スタッフ自身に委ねられる設計・価格)
- Office・メール基盤の構築や販売管理システム等の業務システム開発は
  **本プロジェクトの範囲外**(別事業として時給1万円で提供。
  本書の料金表には含めない)
- **開発費: 無償**(最初の導入店には事例利用を条件に)
- 契約・請求はすべてアプリ外(利用契約書+請求書・銀行振込)。
  アプリは shops.is_active を見るだけで、決済コードは一切書かない
- AGPL/MIT 公開により自力ホストの自由は残る。それで良い。利用料の対価は
  運営中のコミュニティ(利用者・品種マスタ・辞典)への出店である

## 責任分界

- **運営者の責任**: seed と aiseed(運営者のサービス。サーバー・運用)
- **店舗の責任**: 店舗のデータの内容と利用。自力運用する shop の
  サーバー・運用。店舗が自営するメール・OnlyOffice 等の基盤の所有と運用。
  在庫の正(店側在庫API)と在庫管理
- 運用支援・保守は「支援」であり、最終的な運用責任は店舗にある。
  保証範囲・対応時間は契約書に明記する(24時間対応は約束しない)

## リポジトリ構成上の扱い

- 公開リポジトリ: backend, db, packages/core, apps/seed,
  apps/shop, admin, docs
- 非公開リポジトリ: aiseed のみ(core を git 依存で参照)
- SPDX ヘッダ: backend と各 AGPL アプリは `AGPL-3.0-only`、
  core と admin は `MIT`、非公開側は `LicenseRef-Proprietary`
- admin/(MIT)は backend(AGPL)を import するが、Python ソースとして
  配布する限り admin 自体は MIT のままでよい

## 辞典コンテンツ(CC BY-SA 4.0)

- 目的: 辞典を特定サービスへの囲い込みから守る。アプリが消えても
  記事は自由に引き継げる(Wikipedia と同じ構え)
- 利用規約に「投稿された記事・写真は CC BY-SA 4.0 で公開されることに
  同意する」を明記。投稿画面にも一文表示する
- GET /articles/export(認証不要)で実際に持ち出せる状態を保つ
